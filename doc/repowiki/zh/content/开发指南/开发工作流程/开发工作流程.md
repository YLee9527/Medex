## 简介
本指南面向 Medex 项目的开发团队，系统性地梳理从代码审查、版本控制、持续集成到团队协作与开发效率提升的全流程规范。结合现有仓库中的开发文档、发布指南、CI 工作流与技术配置，以及新增的版本管理和部署自动化脚本，形成可落地的开发工作流程，帮助新成员快速上手并维持高质量交付。

## 项目结构
Medex 采用前端（React + TypeScript + Vite）与桌面后端（Tauri + Rust）的混合架构，核心目录与职责如下：
- 前端源码：src/（组件、容器、上下文、页面、主题、状态）
- 桌面后端：src-tauri/（Rust 项目、Tauri 配置、能力与图标）
- 构建与工具：package.json、vite.config.ts、tsconfig.json、tailwind.config.ts
- CI/CD：.github/workflows/main.yml
- 文档：README.md、DEVELOPMENT.md、RELEASE_GUIDE.md、API_REFERENCE.md、THEME_GUIDE.md
- **新增**：版本管理脚本：scripts/（版本同步与部署自动化）

## 架构总览
Medex 的前后端通过 Tauri 的 invoke 与事件通道通信，前端负责 UI 与交互，后端负责文件扫描、数据库与缩略图生成。CI 在打上版本标签时自动触发发布流程。**新增**的版本管理脚本确保了版本号的一致性和发布流程的自动化。

```mermaid
sequenceDiagram

participant VM as "版本管理脚本"
participant FE as "前端应用"
participant Tauri as "Tauri 桌面桥"
participant BE as "Rust 后端"
participant DB as "SQLite 数据库"
Dev->>VM : 修改 version.properties
VM->>VM : 同步版本到所有配置文件
VM->>Dev : 版本同步完成
Dev->>FE : 触发扫描/筛选/打标等操作
FE->>Tauri : invoke("scan_and_index"/"filter_media"/...)
Tauri->>BE : 调用注册的命令
BE->>DB : 读写数据库/事务
BE-->>Tauri : 返回结果/发出事件
Tauri-->>FE : 事件回调进度/缩略图就绪
FE-->>Dev : 更新 UI/状态
```

## 详细组件分析

### 代码审查流程
- Pull Request 模板
  - 建议在 .github/PULL_REQUEST_TEMPLATE.md 中定义 PR 模板，包含变更摘要、影响范围、测试验证、截图/录屏、回归检查清单等。
- 代码审查清单
  - 代码风格与类型安全：TypeScript 严格模式、ESLint（如已配置）、组件函数式写法、Tailwind 使用
  - 交互与性能：虚拟列表、缩略图并发与去重、事件总线与状态同步
  - 安全与权限：Tauri 能力配置、资源协议作用域、外部二进制分发
  - 可测试性：命令与事件的可测试性、错误处理一致性
- 合并策略
  - 采用快进合并或 squash 合并，保留清晰的提交历史；重要变更建议开启二次审查
  - 合并前确保 CI 通过、文档更新、必要时进行小规模回归验证

### 版本控制最佳实践
- 分支策略
  - main：稳定发布线
  - release/*：发布候选分支
  - feature/* / fix/*：功能与修复分支
- 提交消息规范
  - 建议采用约定式提交（feat/fix/docs/chore/style/refactor/test/build/ci），并在 PR 合并时 squash
- 标签管理与版本发布
  - 使用语义化版本（vX.Y.Z），CI 在 push 标签时触发发布
  - 发布前检查：前端构建、Rust 检查、外部二进制存在性
- **更新**：版本号同步与自动化发布
  - **集中版本管理**：使用 version.properties 作为单一版本来源
  - **自动同步**：通过 sync-version.js 和 sync-version.sh 同步到 package.json、tauri.conf.json 和 Cargo.toml
  - **自动化部署**：通过 deploy.js 完成版本号读取、同步、Git 操作和标签创建的完整流程
  - **版本号格式**：遵循语义化版本（MAJOR.MINOR.PATCH），如 1.0.0

## 故障排除指南
- dialog.open 权限
  - 检查 capabilities/default.json 是否包含所需权限
- 本地文件无法预览
  - 确保使用 convertFileSrc 转换文件路径
- 缩略图失败
  - 检查 ffmpeg 是否存在与可执行权限
- 页面卡顿/白屏
  - 排查是否误挂载多个视频、是否启用虚拟化、并发是否过高
- 发布失败
  - externalBin 缺失对应平台二进制、图标缺失或格式不正确
- **新增**：版本管理故障排除
  - **版本文件缺失**：确认 version.properties 文件存在且可读
  - **同步失败**：检查 Node.js 版本和文件权限
  - **部署脚本错误**：查看详细错误信息，确认 Git 配置
  - **CI/CD 失败**：检查 GitHub Secrets 配置和网络连接
