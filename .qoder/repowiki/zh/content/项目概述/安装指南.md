## 简介

Medex 是一个基于 React + TypeScript + Tauri V2 + Rust + SQLite 的桌面媒体管理应用。该项目支持视频和图片的展示、标签管理、分类筛选等功能。项目采用现代化的技术栈，包括三栏式布局（Sidebar / Main / Inspector）和高性能的媒体列表渲染。

## 环境要求

### 核心技术栈要求

根据项目配置，Medex 需要以下环境和技术栈：

- **Node.js**: 18+ 版本
- **Rust**: 1.77.2+ 版本
- **包管理器**: npm 或 pnpm
- **构建工具**: Vite 5.4
- **前端框架**: React 18.3 + TypeScript 5.5
- **样式框架**: TailwindCSS 3.4 + PostCSS + Autoprefixer
- **状态管理**: Zustand 4.5
- **桌面框架**: Tauri V2

### 外部运行时依赖

- **ffmpeg**: 用于视频首帧缩略图生成（可选，项目提供回退机制）

## 安装步骤

### 步骤 1: 安装 Node.js 和包管理器

#### Windows/macOS/Linux 通用安装方法

1. **安装 Node.js 18+**
   - 访问 [Node.js 官方网站](https://nodejs.org/)
   - 下载 LTS 版本（推荐）
   - 按照安装向导完成安装

2. **验证 Node.js 安装**
   ```bash
   node --version
   npm --version
   ```

3. **选择包管理器**
   - 推荐使用 npm（Node.js 默认）
   - 或者安装 pnpm 作为替代方案

**章节来源**
- [README.md:54-56](file://README.md#L54-L56)

### 步骤 2: 安装 Rust 工具链

#### Windows 安装方法

1. **下载安装 Rust**
   - 访问 [rustup.rs](https://rustup.rs/)
   - 下载 rustup-init.exe
   - 运行安装程序并选择默认选项

2. **验证 Rust 安装**
   ```bash
   rustc --version
   cargo --version
   ```

#### macOS 安装方法

1. **使用 Homebrew**
   ```bash
   brew install rustup
   rustup-init
   ```

2. **手动安装**
   ```bash
   curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
   ```

#### Linux 安装方法

1. **使用官方脚本**
   ```bash
   curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
   ```

2. **添加到 PATH**
   ```bash
   source ~/.cargo/env
   ```

## 操作系统特殊注意事项

### Windows 特殊注意事项

1. **Rust 工具链**
   - 确保安装了 MSVC 工具链
   - 运行: `rustup toolchain install stable`
   - 运行: `rustup target add x86_64-pc-windows-msvc`

2. **构建依赖**
   - 安装 Windows SDK
   - 确保 Visual Studio Build Tools 已安装

3. **权限要求**
   - 首次运行可能需要管理员权限
   - 防病毒软件可能需要例外

### macOS 特殊注意事项

1. **Xcode 命令行工具**
   ```bash
   xcode-select --install
   ```

2. **Homebrew 依赖**
   ```bash
   brew install rustup
   rustup-init
   ```

3. **签名和公证**
   - 开发模式无需签名
   - 发布版本需要 Apple Developer 账户
   - 使用 `codesign` 进行签名

### Linux 特殊注意事项

1. **系统依赖**
   ```bash
   # Ubuntu/Debian
   sudo apt update
   sudo apt install build-essential libgtk-3-dev libwebkit2gtk-4.0-dev
   
   # CentOS/RHEL/Fedora
   sudo yum groupinstall "Development Tools"
   sudo yum install gtk3 webkit2gtk3-devel
   
   # Arch Linux
   sudo pacman -S base-devel webkit2gtk
   ```

2. **Rust 工具链**
   ```bash
   curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
   source ~/.cargo/env
   ```

3. **桌面环境**
   - 确保安装了 GTK3 和 WebKit2GTK
   - 某些发行版可能需要额外的字体包

## 常见问题解决

### 问题 1: Node.js 版本不兼容

**症状**: 安装依赖时报错
**解决方案**:
```bash
# 检查 Node.js 版本
node --version

# 升级到 LTS 版本
# 重新安装项目依赖
npm install
```

### 问题 2: Rust 编译错误

**症状**: `cargo build` 失败
**解决方案**:
```bash
# 更新 Rust 工具链
rustup update

# 清理构建缓存
cargo clean

# 重新构建
cargo build
```

### 问题 3: ffmpeg 未找到

**症状**: 缩略图生成失败
**解决方案**:
```bash
# 安装 ffmpeg
# Windows: choco install ffmpeg
# macOS: brew install ffmpeg
# Linux: sudo apt install ffmpeg

# 验证安装
ffmpeg -version

# 或者将 ffmpeg 放置在 src-tauri/binaries/ 目录
```

**章节来源**
- [DEVELOPMENT.md:564-586](file://DEVELOPMENT.md#L564-L586)

### 问题 4: Tauri 开发模式启动失败

**症状**: `npm run tauri dev` 报错
**解决方案**:
```bash
# 检查前端构建
npm run build

# 检查 Rust 依赖
cd src-tauri
cargo check

# 清理并重新构建
cargo clean
cargo build
```

### 问题 5: 端口占用

**症状**: 开发服务器无法启动
**解决方案**:
```bash
# 检查端口占用
netstat -ano | findstr :1420

# 修改端口配置
# 在 vite.config.ts 中修改 port: 1420
```

## 故障排除指南

### 构建相关问题

#### 问题: 构建失败，提示缺少依赖

**诊断步骤**:
1. 检查网络连接
2. 验证 Cargo 配置
3. 清理缓存并重新安装

**解决方案**:
```bash
# 清理 Cargo 缓存
cargo clean
rm -rf src-tauri/target

# 重新安装依赖
cd src-tauri
cargo update

# 返回项目根目录
cd ..
npm install
```

#### 问题: 跨平台构建失败

**诊断步骤**:
1. 检查目标平台支持
2. 验证交叉编译工具链
3. 检查系统依赖

**解决方案**:
```bash
# 安装交叉编译工具
rustup target add x86_64-pc-windows-gnu
rustup target add x86_64-unknown-linux-gnu

# 安装系统依赖
# Windows: 安装 MinGW
# Linux: 安装 g++-multilib
```
